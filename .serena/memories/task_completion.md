# タスク完了前チェック
- コード変更後は `npm run lint` を必ず実行。型影響があれば `npm run typecheck` も併用。
- 仕様/ロジック追加時は Jest テストを更新・追加し、`npm test` で確認。UI 追加は手動で `/` を開きライト/ダークとナビゲーションを確認。
- 大きな変更や依存追加時は `npm run build` で本番ビルド確認を推奨。
- Supabase 関連変更時は `.env.local` を再確認し、必要なら型生成を検討・説明を README に追記。
- PR 前は変更点を簡潔に記述し、UI 変更ならスクリーンショット/GIF を用意。
